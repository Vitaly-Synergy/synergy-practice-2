# Кейс-задача №2: Демонстрация наследования и полиморфизма в C++

## Описание

Тестовая программа, демонстрирующая работу методов базового и производных классов в C++. Реализована система учёта сотрудников с использованием механизмов наследования, переопределения методов и полиморфизма.

## Цель работы

Показать на практике:
- Создание иерархии классов (базовый и производные классы)
- Переопределение виртуальных методов (`override`)
- Вызов методов базового класса из производного
- Полиморфное поведение через указатели на базовый класс
- Безопасную работу с памятью (виртуальный деструктор, `unique_ptr`)
- Интерактивный ввод данных от пользователя

## Архитектура классов

```
Employee (базовый класс)
    ├── name: string
    ├── work(): virtual void
    └── getName(): string
    
    ↓ наследование
    
    ├── Developer (производный класс)
    │   ├── language: string
    │   ├── work(): override void
    │   └── writeCode(): void
    │
    └── Manager (производный класс)
        ├── work(): override void
        └── holdMeeting(): void
```

## Демонстрируемые механизмы C++

1. **Наследование** — `Developer` и `Manager` наследуются от `Employee` через `public`
2. **Виртуальные методы** — `virtual void work()` в базовом классе
3. **Переопределение** — ключевое слово `override` в производных классах
4. **Вызов базового метода** — `Employee::work()` внутри производных классов
5. **Полиморфизм** — хранение объектов в `vector<unique_ptr<Employee>>` и вызов `work()` через указатель на базу
6. **Виртуальный деструктор** — `virtual ~Employee() = default;` для предотвращения утечек памяти
7. **Приведение типов** — `dynamic_cast` для вызова уникальных методов производных классов
8. **Безопасный ввод** — `std::getline` для корректной работы со строками, содержащими пробелы
9. **Умные указатели** — `std::unique_ptr` для автоматического управления памятью

## Сборка и запуск

### Требования
- Компилятор C++ с поддержкой C++17 (GCC, Clang, MSVC)
- CMake (опционально)

### Вариант 1: Прямая компиляция

**Linux / macOS / WSL:**
```bash
g++ -std=c++17 -Wall -Wextra src/main.cpp -o demo
./demo
```

**Windows (MSVC):**
```cmd
cl /EHsc /std:c++17 src\main.cpp
demo.exe
```

### Вариант 2: CMake

```bash
mkdir build && cd build
cmake ..
cmake --build .
./demo
```

## Особенности реализации

- **UTF-8 поддержка**: программа корректно работает с кириллицей (на Windows используется `chcp 65001`)
- **Безопасность памяти**: используются умные указатели `std::unique_ptr` вместо сырых указателей
- **Валидация ввода**: функция `readLine()` обеспечивает безопасное чтение строк с пробелами
- **Константная корректность**: методы, не изменяющие состояние объекта, помечены как `const`
